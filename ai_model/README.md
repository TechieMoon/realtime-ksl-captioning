# Local MediaPipe MVP Model Package

This folder keeps the local Hugging Face-compatible MediaPipe MVP package used by
the legacy training scripts. The current end-to-end app uses the team model at
`Seoyoung07/korean-sign-word-classifier-mediapipe`; this local package remains
only for reproducing or uploading the older MediaPipe MVP artifact.

The trained `mediapipe_mvp.joblib` artifact is intentionally not tracked in
GitHub. If that artifact is absent, `inference.py` returns empty predictions so
backend contract tests can run without a local checkpoint.

## Runtime Contract

```python
def load_model(model_dir: str, device: str):
    ...

def predict(model, frames: list, timestamps_ms: list[int | None]) -> list[dict]:
    ...
```

Each frame must be an RGB `numpy.ndarray` with shape `(height, width, 3)` and
dtype `uint8`.

Prediction output:

```json
{
  "text": "안녕하세요",
  "words": [
    {
      "text": "안녕하세요",
      "confidence": 0.91,
      "start_ms": 1200,
      "end_ms": 1700
    }
  ],
  "is_final": true
}
```

## Kept Code

- `inference.py`: backend/Hugging Face contract entry point.
- `mediapipe_mvp.py`: MediaPipe keypoint extraction and lightweight sequence
  classifier runtime.
- `labels.json`: vocabulary file generated by the legacy training flow.
- `requirements.txt`: dependencies for this local MediaPipe MVP package.

Unused CTC/gloss decoder code is not part of this GitHub package. The
untrained YOLO/VideoMAE scaffold was removed so the repository reflects the
code paths that are actually used.
